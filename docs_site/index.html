Platform Summary

What the platform is built to do

NDU Project is an enterprise project lifecycle platform spanning initiation, planning, design, execution, launch, governance, and close-out across project, program, and portfolio contexts.

Lifecycle

How work flows from idea to closure

The platform is organized as a guided operating system for project delivery, with structured outputs at every phase and deep planning support through the Front-End Planning suite.

Capability Domains

Coverage across planning, delivery, intelligence, and operations

These are the major product capability layers reflected in routes, services, widgets, and operational docs.

Route Atlas

Screen inventory by product cluster

This atlas mirrors the route taxonomy defined in the Flutter router so implementation and documentation stay aligned.

Architecture

How the platform is composed

Flutter drives the client experience, Firebase powers identity and persistence, and service modules coordinate AI, workflow automation, payments, and operational tooling.

Data Model

Project state, collections, and persistence model

The platform revolves around a large shared ProjectDataModel plus Firestore subcollections for execution records, contracts, vendors, issues, stories, and operational artifacts.

KAZ AI

AI workflows, safety model, and usage boundaries

KAZ AI is used to accelerate planning, summarization, recommendations, diagram generation, and structured content drafting while protecting secrets through a server-side proxy architecture.

Commercial Stack

Billing, subscriptions, coupons, and entitlements

The platform supports multi-provider billing, tiered subscriptions, invoice history, coupon flows, and subscription-aware access experiences.

Admin and Governance

Operational control surfaces

The admin environment provides user operations, subscription lookup, coupon management, content controls, and restricted host-aware routing.

Integrations

External systems, OAuth, and service boundaries

Beyond Firebase and payments, the platform is structured to support collaboration and whiteboarding integrations, secure key handling, and workflow-linked external tooling.

Deployment

Firebase Hosting and operational rollout

The docs site is static and Hosting-ready. The wider platform also depends on Firestore rules, Cloud Functions, secrets, and environment configuration for AI and payments.
