UI Principles

Common page anatomy across the platform

Most NDU pages share a workspace model: a strong title/header region, sectional content panels, contextual actions, persistence-aware forms, and side navigation that keeps the user oriented inside a long lifecycle journey.

Planning pages usually combine rich cards, guided text areas, AI generation controls, scenario matrices, and summary blocks.

These pages lean toward specification structure, interface records, technical notes, tables, and engineering readiness artifacts.

Execution and Operations

How live-delivery pages are composed

Operational pages move away from narrative forms and toward boards, logs, tracked records, quick actions, and real-time tables.

Admin UI

Back-office and control-surface pages

The admin runtime emphasizes oversight: lookup, filtering, editable content, management tables, and trust-oriented operational actions.
